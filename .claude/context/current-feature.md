# Current Feature

AI-generated event description

## Summary

When we click an event and open the modal, many of the events lack descriptions. And when they do have descriptions, it often is only details about age restrictions, and entry time.

We want to utilise our OpenAI SDK to generate a description of the event itself.

- The description should include who or what is the main act of the event.
- If applicable, also describe the vibe of the event (eg. slow moody jazz atmosphere)
- We must let the user know that the description was generated by AI, eg. "Description generated by OpenAI"
- Show a loading state whilst the description attempts to generate.
- Cache the descriptions if generated.
- If OpenAI fails to generate a description for some reason eg.  past free plan usage, can't identify the specific details of the event, it should have some placeholder text: "Could not generate details of the event - go to the event page to learn more."

Reference the Event Data section in the .claude/context/project-overview.md to get an idea of what data we already have available to work with, in order to generate an AI description for the event.

## Requirements

- Create a new API route (e.g. `/api/openai/describe-event`) that accepts event data and returns an AI-generated description
- Send event name, venue name, date, and existing `info`/`pleaseNote` fields to `gpt-4o-mini` as context for generation
- The description should focus on: who/what the main act is, and optionally the vibe/atmosphere of the event
- Display the AI description in the EventModal (`eventModal.tsx`), in the "About this event" section
- Show a loading state (spinner or skeleton) while the description is being generated
- Display an attribution label: "Description generated by OpenAI"
- On failure (API error, rate limit, unidentifiable event), show fallback text: "Could not generate details of the event - go to the event page to learn more."
- Cache generated descriptions client-side (e.g. in state or context keyed by event ID) so re-opening the same event doesn't re-call the API
- Only trigger AI generation when the modal opens and the event lacks a meaningful `info` field (or `info` is just venue/entry details)

## Notes

- Existing OpenAI integration lives in `next/src/pages/api/openai/classify-event.ts` and `next/src/helpers/openai.ts` — follow the same pattern
- Model: `gpt-4o-mini` (already used for suggestions, very cheap at ~$0.15/1M input tokens)
- Available event data from Ticketmaster: `name`, `_embedded.venues[0].name`, `dates.start.localDate`, `dates.start.localTime`, `info`, `pleaseNote`, `_embedded.attractions`
- The EventModal is in `next/src/components/Modal/eventModal.tsx` — currently shows `gig.info` under "About this event" only when it exists
- Free plan risk: rate limit is ~3 RPM on free tier; credits expire after 3 months. Caching is important to minimize calls
- The `Gig` type is defined in `next/src/types.ts`
