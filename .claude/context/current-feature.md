# Improve Suggestion Modal UI/UX

## Summary

As soon as the page loads, a modal appears that allows you to select a mood and get some recommended gigs based on the mood. You can click one of several buttons with a label eg. "Lots of lights and a boogie"...

It does not have the most eye-catching UI/UX. The most critical aspect is that it is not engaging enough to the eye to differentiate between each suggestion button. It just looks like a boring list, with not enough contrast in terms of colour, hierarchy, etc.

- We should update the layout of the buttons in a 2 column grid, where the buttons look more square.
- Each button should have some better contrast from the modal background colour, and an emoji to match each label.
- Ensure the other main elements of the modal - title, close button and datepicker - are re-arranged in a way appropriate to the layout, hierarchy and overall UX of the modal.
- Fix up any awkward/unbalanced spacing.

## Requirements

- Lay out the suggestion buttons in a 2-column grid, with each button shaped more like a square.
- Give each button stronger contrast against the modal background, and add an emoji that matches the label.
- Rearrange the modal's other main elements (title, close button, datepicker) to suit the new layout and improve hierarchy/UX.
- Fix any awkward or unbalanced spacing throughout the modal.

## Notes

- The modal appears automatically on page load, so it is effectively the user's first impression of the app.
- Core UX problem: suggestion buttons currently read as a boring list — insufficient contrast, weak visual hierarchy, no differentiation between options.
- Scope is UI/UX of the existing modal only; suggestion logic / AI recommendations themselves are unchanged.
